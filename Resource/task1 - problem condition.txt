Реализовать Буфер, Лексер, Парсер. Покрыть их тестами.
Начать стоит с реализации только арифметических выражений.

Упрощённые правила вывода для первого задания:
expr -> term [+-] term [+-] ... [+-] term // неформально определены
term -> factor [*/] factor [*/] ... [*/] factor // неформально определены
factor -> power ^ factor | power
power -> atom | - atom
atom -> number | ( expr )

Некоторые детали реализации:
1. Вещественнае числа всегда вида "35.053",
    т.е. всегда есть целая часть(может быть 0) и дробная(может быть нулевая).
2. Все числа считаются сразу в double, чтобы пока не париться
3. Алгоритм сборки вещественного числа:
    считываем в int все цифры, запоминаем порядок запятой, делим на этот порядок.
